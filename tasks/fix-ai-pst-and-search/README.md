# 修复 AI 评估函数 PST 翻转错误 + 增强搜索

## 问题描述
1. **深度8崩溃**: `Fatal Python error: _PyEval_EvalFrameDefault: Executing a cache`
2. **深度5走棋蠢**: "三高兵胜士象全"残局中，AI 宁可走仕也不推兵吃象

## 根因
### PST 位置加成表查表方向完全反了 (`alpha_beta.py` evaluate())
PST 表含义: `PST[0]` = 深入对方阵地(高分), `PST[9]` = 己方底线(低分)

- **红方** (底线 grid row9, 深入 grid row0):
  - 错误: `pst[9 - r]` → 深入时查底线低分
  - 正确: `pst[r]` → 深入时查深入高分
- **黑方** (底线 grid row0, 深入 grid row9):
  - 错误: `pst[r]` → 底线时查深入高分
  - 正确: `pst[9 - r]` → 底线时查底线低分

**结果**: 红方和黑方的 PST 完全写反了，导致 AI 认为棋子待在原地比深入进攻好。

### 深度8崩溃
CPython 3.13 JIT/specialization 相关 bug，大量递归调用触发。通过置换表减少搜索节点可缓解。

## 修复内容 (`alpha_beta.py`)
1. **修复 PST 查表方向**: 红方 `pst[r][c]`，黑方 `pst[9-r][c]`
2. **添加置换表 (Transposition Table)**: 缓存已搜索局面，避免重复计算
3. **迭代加深 (Iterative Deepening)**: 从深度1逐步加深到目标深度，浅层结果改善深层排序
4. **改进走法排序**: MVV-LVA + 置换表最佳走法优先
