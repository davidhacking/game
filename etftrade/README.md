# Trade A50 V2 — 单 ETF 量化择时系统

交易标的：**科创50 ETF (SH.588000)**，数据通过 FutuOpenD API 获取，交易通过 SSH → Windows VM → 同花顺 GUI 执行。

---

## 使用

```bash
# 训练
./train.sh                     # 默认 split_date=20250101
./train.sh 20260101            # 自定义分割日期

# 回测
./backtest.sh                  # 默认从 meta.json 中的 split_date 开始
./backtest.sh 20260101         # 指定回测起始日期
./backtest.sh 20260101 SH.588000 500000  # 起始日期 + ETF代码 + 初始资金

# 预测 (三种输入方式)
./predict.sh                                                # 自动从同花顺获取持仓
./predict.sh portfolio.json                                 # 从 JSON 文件读取
echo '{"balance":1000000,"position":{}}' | ./predict.sh     # stdin 传入
./predict.sh --execute                                      # 输出建议 + 自动执行交易

# 获取持仓
./load_position.sh             # 从同花顺获取当前账户信息 (JSON)

# 自动交易 (获取持仓 → 预测 → 执行)
./trade.sh                     # = load_position.sh | predict.sh --execute
```

## 策略设计 (V4)

### 核心思想

**模型做波段**: 用 LGB+XGB+CatBoost 三模型融合预测 ETF 多时间窗口加权收益率，预测涨就持有/加仓，预测跌就卖出观望，等跌完再进。

**模型是唯一决策者** —— 没有硬编码的 MA/RSI/MACD 规则。

### 版本演进

| | V2 | V3 | V4 (当前) |
|---|---|---|---|
| 模型 | 二分类 LGB | 单 LGB 回归 | **LGB+XGB+FT-Transformer 三模型融合** |
| 训练数据 | 1只ETF ~800条 | 71只ETF 17万条 | **90只ETF 20万条 (71只宽基+19只科技)** |
| 预测目标 | 单一5日涨跌 | 单一10日收益率 | **5d/10d/20d/30d 加权融合** |
| 防泄漏 | 无 | purge 7天 (不够) | **purge 35天 (≥最长前瞻30d)** |
| 模型评估 | AUC=0.53 (无效) | 截面IC=0.50 (虚高泄漏) | **时序IC=-0.064 (验证集)** |
| 模型作用 | 装饰 (贡献~10%) | 决策核心 | **决策核心** |

## 训练流程 (train.py)

```
train.sh [split_date] [etf_code]
  │
  ├─ 1. 数据准备 (build_training_data)
  │     90 只 A 股 ETF: 旧版71只 (从FutuOpenD BK0800板块自动筛选) + 新增19只科技行业ETF
  │     FutuOpenD 下载日K (2015~今天), 每只需 ≥500 条数据
  │     每只 ETF 独立做 build_features() → 50 个技术指标
  │     合并为 20万+ 行联合训练集, 缓存到 data/multi_etf_features.pkl
  │
  ├─ 2. 多时间窗口融合标签
  │     对每只 ETF 计算 4 个前瞻收益率:
  │       ret_5d_fwd  = close[t+5]  / close[t] - 1
  │       ret_10d_fwd = close[t+10] / close[t] - 1
  │       ret_20d_fwd = close[t+20] / close[t] - 1
  │       ret_30d_fwd = close[t+30] / close[t] - 1
  │     加权融合为单一标签:
  │       fused_target = 0.15×ret_5d + 0.30×ret_10d + 0.30×ret_20d + 0.25×ret_30d
  │     权重可在 config.py 的 HORIZON_WEIGHTS 中配置
  │
  ├─ 3. 三模型融合训练 (train_model)
  │     数据划分 (purged time-series split, 防未来泄漏):
  │       训练集: split_date 前8个月之前 (purge 35天 gap, ≥ 最长前瞻30天)
  │       验证集: split_date 前8个月 ~ split_date (purge 35天 gap)
  │       详见下方「数据划分详解」
  │
  │     三个模型独立训练, 预测时取均值:
  │       ┌────────────────┬──────────────────────────────────────────────┐
  │       │ LightGBM       │ GBDT, lr=0.01 leaves=63 depth=7 L1=5 L2=20  │
  │       │ XGBoost        │ GBDT, lr=0.01 depth=7 alpha=5 lambda=20      │
  │       │ FT-Transformer │ Transformer, dim=32 depth=4 heads=8          │
  │       └────────────────┴──────────────────────────────────────────────┘
  │       prediction = (lgb_pred + xgb_pred + cb_pred) / 3
  │
  │     参数/模型选择/融合方式详见下方「三模型融合详解」
  │     早停: 验证集 MAE 连续500轮没进步就停, 最多3000轮
  │     (详见下方「早停机制」)
  │
  ├─ 4. 预测值分布统计
  │     在训练集上做融合预测, 记录分布分位数:
  │       p10/p25/p50/p75/p90 → 存入 meta.json 的 pred_stats
  │     推理时把预测值映射到仓位 (详见 pred_to_position)
  │
  ├─ 5. 模拟交易 (simulate_trading)
  │     在目标 ETF 上从 split_date 模拟到今天, 初始资金100万
  │     每天: 三模型融合预测 → 5日SMA平滑 → 分位数映射 → 交易
  │     计算: 策略收益/胜率/赔率 vs Buy&Hold
  │
  └─ 6. 保存
        models/model.pkl       — LGB+XGB 打包: {'lgb': Booster, 'xgb': XGBRegressor}
        models/ft_transformer.pt — FT-Transformer 权重 (PyTorch, 单独保存)
        models/meta.json  — 元数据 (IC/MAE/pred_stats/特征列/配置等)
```

### 时序 IC 是什么 (大白话)

**一句话**: 模型每天的预测和实际涨跌有多"像"。

举个例子, 模型今天对科创50的预测是 +0.2%, 对银行ETF的预测是 -0.1%。
10天后实际结果: 科创50涨了1%, 银行ETF跌了0.5%。
模型说"科创50比银行ETF好" → 实际也是 → 这一天的IC就是正的。

**截面 IC vs 时序 IC**:

```
截面 IC: "同一天, 模型对90只ETF的排名准不准"
         → 天然偏高, 因为不同行业差异大, 很容易排对
         → 类比: 期末考试排全年级名次, 差生和优等生很容易区分

时序 IC: "同一只ETF, 模型对每天涨跌的预测准不准"
         → 这才是择时需要的能力
         → 类比: 预测同一个学生明天考得比今天好还是差, 难得多
```

参考值:
- IC < 0.03 → 基本没用, 和瞎猜差不多
- IC 0.03~0.05 → 能用了, 量化基金单因子的常见水平
- IC 0.05~0.10 → 不错
- IC > 0.10 → 非常好
- **IC > 0.30 → 几乎一定有bug** (数据泄漏/过拟合)

当前模型验证集上的时序IC偏低, 但测试集上策略跑赢 Buy&Hold, 说明:
- 模型学到了微弱但有用的信号
- 仓位管理 (分位数映射+平滑) 把微弱信号放大了
- 验证集时段 (2024年) 恰好是市场较难预测的阶段

### 数据划分详解 (大白话)

**问题**: 模型要预测"未来30天的涨跌", 但训练时不能偷看未来的数据。

假设 split_date = 2025-01-01 (从这天开始实盘):

```
时间轴:

  2015 ─────────── 2024-03 ── 2024-04 ─── 2024-05 ──────── 2024-11 ── 2025-01
   │                  │          │            │                │          │
   │    训练集         │ 35天隔离 │  验证集     │                │ 35天隔离 │ 实盘开始
   │  (模型学习用)     │  (空白)  │ (模型考试用) │                │  (空白)  │
   └──────────────────┘          └────────────────────────────┘          │
                                                                         ▼
                                                              这天之后的数据用来实际交易
```

三段数据的作用:

  训练集 (2015 ~ 2024-03):  模型拿这些数据"做题"学规律
  验证集 (2024-05 ~ 2024-11): 模型拿这些数据"模拟考试", 看学得好不好, 决定什么时候停止训练
  测试集 (2025-01 ~):        实盘/回测, 模型从没见过的数据

**为什么中间要留35天空白 (purge gap)?**

因为标签用到了未来30天的价格:
  fused_target = 0.15×ret_5d + 0.30×ret_10d + 0.30×ret_20d + 0.25×ret_30d

如果训练集最后一天是2024-03-31, 它的标签包含2024-04-30的收盘价。
验证集从2024-04-01开始, 它的特征包含2024-04-01的收盘价。
2024-04-30和2024-04-01的收盘价高度相关 → 信息泄漏!

所以中间必须空出 ≥30天 (我们用35天留余量):
  训练集截止 2024-03-26 → 标签用到 2024-04-25 的价格
  验证集从   2024-05-01 开始 → 特征是 2024-05-01 的价格
  之间隔了35天, 不会有价格重叠 → 没有泄漏

**类比**: 就像老师出期中考试卷, 不能从学生刚做过的作业里抄题,
至少要隔一个月, 否则学生"背答案"就能考高分 (虚高的IC)。

### 三模型融合详解

#### 参数含义 (大白话)

**lr (learning_rate) = 0.01 — 学习步长**

模型每轮学一棵新树来纠正上一轮的错误。lr 控制每棵树的影响力:
- lr=1.0: 每棵树全力纠错 → 学得快但容易矫枉过正 (震荡)
- lr=0.01: 每棵树只纠正1%的错误 → 学得慢但更稳, 需要更多轮才收敛
- 类比: 调水龙头温度, lr大=大幅拧 (忽冷忽热), lr小=微调 (慢慢到合适温度)

**depth (max_depth) = 7 — 树的最大深度**

决策树通过一层层 if-else 做判断:
```
第1层: RSI > 70?
  ├─ 是 → 第2层: MACD < 0?
  │         ├─ 是 → 第3层: 近5日跌幅 > 3%?
  │         │         ├─ 是 → 预测: 大跌 -2%
  │         │         └─ 否 → 预测: 小跌 -0.5%
  │         └─ 否 → ...
  └─ 否 → ...
```
depth=7 表示最多嵌套7层判断。太浅 (3层) 学不到复杂规律,
太深 (15层) 会把噪声也当规律记住。7层是经验平衡点。

**num_leaves = 63 — 叶节点数 (LightGBM特有)**

一棵树最终有多少个"结论"。depth=7 理论上最多 2^7=128 个叶子,
但设成63约一半, 相当于限制树不能长满, 留有余地。

**L1 = 5, L2 = 20 — 正则化 (防过拟合的"刹车")**

```
没有正则化: 模型自由发挥 → 把训练数据背得滚瓜烂熟 → 新数据上表现差
L1 正则化:  惩罚"用了太多特征" → 逼模型只用最重要的几个特征, 其余权重→0
L2 正则化:  惩罚"权重太大" → 逼模型对每个特征只给小权重, 不要太依赖单一特征

类比:
  没有正则化 = 学生可以带所有资料进考场 → 背答案就行
  L1 = 只能带3本书 → 必须挑最有用的
  L2 = 每本书只能翻5分钟 → 不能只靠一本书
```
金融数据噪声极多, 所以 L1=5, L2=20 都偏大 (刹车踩得重)。

**min_child_samples = 50 — 最小样本数**

每个叶节点 (最终结论) 至少要有50个样本支撑。
如果只有3个样本说"RSI>80且MACD<-0.5时暴跌", 可能只是巧合。
50个样本说同样的话, 才更可能是真规律。

**subsample = 0.8, colsample_bytree = 0.6 — 随机采样**

```
subsample=0.8:      每棵树随机抽80%的数据来学 (留20%不看)
colsample_bytree=0.6: 每棵树随机抽60%的特征来用 (其余不看)
```
好处: 每棵树看到的数据和特征都不完全一样 → 树之间有差异 → 合在一起更稳。
类比: 让100个分析师各看不同的指标子集, 比让100个人看同样的数据, 合在一起结论更靠谱。

```
三个模型的参数对照:
  ┌──────────────┬──────────┬──────────┬──────────┐
  │    参数      │ LightGBM │ XGBoost  │ CatBoost │
  ├──────────────┼──────────┼──────────┼──────────┤
  │ 学习率       │ 0.01     │ 0.01     │ 0.01     │
  │ 树深度       │ 7        │ 7        │ 7        │
  │ L1正则化     │ 5        │ 5        │ -        │
  │ L2正则化     │ 20       │ 20       │ 20       │
  │ 最小样本数   │ 50       │ 50       │ -        │
  │ 数据采样     │ 80%      │ 80%      │ 80%      │
  │ 特征采样     │ 60%      │ 60%      │ 60%      │
  │ 叶节点数     │ 63       │ -        │ -        │
  └──────────────┴──────────┴──────────┴──────────┘
  三个模型参数尽量对齐, 差异只在各自特有的控制方式上
```

一句话: **所有正则化参数都偏大 (L1=5, L2=20, min_samples=50)**,
因为金融数据噪声极多, 宁可让模型"学少一点但更稳", 也不要"学多了但全是噪声"。

#### 为什么选这三个模型

三个都是 GBDT (梯度提升树) 家族, 但实现方式不同:

| 模型 | 类型 | 特点 |
|------|------|------|
| LightGBM | GBDT | 按叶子生长, **最快**, 实测表现最稳 |
| XGBoost | GBDT | 按层生长, **最成熟**, 正则化最精细 |
| FT-Transformer | Transformer | 注意力机制学特征交互, **和GBDT互补** |

FT-Transformer 替代了原来的 CatBoost (CB只训0轮, 没贡献)。
Transformer 和 GBDT 是完全不同的模型架构, 融合后能学到 GBDT 学不到的特征关系。

**为什么不用其他模型?**

| 模型 | 不用的原因 |
|------|-----------|
| 随机森林 | 同样是树模型但没有"提升"(逐步纠错), 表格数据上通常不如GBDT |
| 线性回归 | 太简单, 无法学到非线性关系 (如"RSI>70且MACD死叉→跌") |
| LSTM/Transformer | 需要更多数据+更长调参, 7万~20万行还不够, 且容易过拟合 |
| SVM | 大数据量下训练极慢, 且超参数敏感 |

三个 GBDT 变体是**结构化数据 (表格) 上的最强组合**, Kaggle 竞赛中常年霸榜。

#### 为什么平权融合 (等权平均)

```python
prediction = (lgb_pred + xgb_pred + cb_pred) / 3
```

**平权而不加权的原因:**

1. **简单最稳**: 加权融合需要额外的验证集来调权重, 容易过拟合权重本身
2. **三个模型结构相似**: 都是 GBDT, 预测能力接近, 没有明显的"强弱之分"
3. **理论支持**: 当模型误差不相关时, 等权平均已经是最优融合
   (降方差: 方差降到 1/3, 不需要调权重)
4. **实测验证**: 当前 XGB/CB 早停太快 (6轮/0轮), 主要靠 LGB,
   但平权取均值至少不会比单 LGB 差——最坏情况等于 LGB 被稍微拉平

**什么时候值得加权?**
当三个模型差异很大 (如树模型+线性模型+神经网络) 时,
用验证集上的表现反推权重才有意义。目前三个 GBDT 没必要。

### 早停机制 (Early Stopping)

**一句话**: 模型每多学一轮就在"考试卷"(验证集)上测一次分, 连续500轮分数没进步就停。

```
训练过程 (以 LightGBM 为例):

  第1轮:  模型学了1棵树  → 验证集 MAE=0.080 (误差大, 还很菜)
  第10轮: 已经10棵树     → MAE=0.065 (进步中)
  第30轮: 30棵树         → MAE=0.057 ← 最好成绩! ★
  第31轮: 31棵树         → MAE=0.058 (反而变差了, 开始"背答案")
  ...
  第530轮: 连续500轮都没超过第30轮的 0.057
  → 触发早停! 回退到第30轮的模型
```

**为什么要早停**: 防止模型"背答案"(过拟合)。
树越多, 模型在训练数据上越准, 但在新数据上可能越差。
就像学生只背往年真题, 遇到新题就不会了。

**当前参数**: max=3000轮, patience=500轮, lr=0.01。
LGB 实际在第30轮左右最佳, XGB/CatBoost 更早。
说明特征中可提取的信号有限, 模型很快就学完了有用信息。

## 仓位映射 (pred_to_position)

模型预测值很小 (±0.3%), 不能直接用绝对值做仓位。
用**训练集预测值分布的分位数**做映射:

```
模型预测 fused_target → 在训练集分布中的位置 → 目标仓位

  pred ≤ p10 (后10%, 强烈看跌) → 0%       清仓观望
  pred p10~p25                  → 15%~30%  轻仓防守
  pred p25~p50                  → 30%~60%  适中持仓
  pred p50~p75                  → 60%~85%  重仓
  pred p75~p90                  → 85%~95%  准满仓
  pred > p90                    → 95%      满仓
```

**5日移动平均平滑**: 对每天的原始预测取最近5天均值, 避免单日噪声导致频繁交易。

**渐进调仓 (指数衰减追踪)**:
```
每天实际调仓 = (目标仓位 - 当前仓位) × 0.3

例: 目标仓位87%, 当前0%
  第1天: 实际买入 (87%-0%) × 0.3 = 26%  → 仓位 26%
  第2天: 实际买入 (87%-26%) × 0.3 = 18% → 仓位 44%
  第3天: 实际买入 (87%-44%) × 0.3 = 13% → 仓位 57%
  第5天: 仓位约 72%
  第7天: 仓位约 83%  → 接近目标

好处: 大调仓分批执行 (降低冲击), 小微调一步到位
```

**最小调仓阈值**: 渐进后实际调仓幅度 < 5% 就不交易, 减少摩擦成本。

## 预测与交易 (predict.py)

### 两类持仓的处理

通过 `config.py` 配置:

```python
BUY_LIST = ['SH.588000']          # 允许买入的标的
SELL_THRESHOLD_PERCENTILE = 'p25'  # 其他持仓卖出阈值
```

| 持仓类型 | 处理逻辑 |
|----------|---------|
| **BUY_LIST 标的** (如 588000) | 三模型融合预测 → 分位数映射 → 目标仓位 → BUY/SELL/HOLD |
| **其他持仓** (如 600000, 600547 等) | 三模型融合预测 → smoothed_pred < p25 → SELL; 否则 HOLD |

### 执行顺序 (--execute 模式)

严格先卖后买, 确保用实际现金计算买入额:

```
Step 1: BUY_LIST 标的减仓 → 如果目标仓位 < 当前仓位, 先卖出多余部分
Step 2: 其他持仓卖出    → 模型预测跌的, 全部卖出 (回笼资金)
Step 3: BUY_LIST 标的加仓 → 用 Step1+2 回笼的资金 + 原有现金, 计算实际可买入额
```

### 股票代码转换

持仓中的代码为纯数字, FutuOpenD 需要带市场前缀:
- `6` 开头 → `SH.6xxxxx` (上海)
- `0` / `3` 开头 → `SZ.0xxxxx` / `SZ.3xxxxx` (深圳)

## 已知限制与后续方向

1. **时序IC偏低**: 50个特征全是传统技术指标, 信息重叠度高。后续可加入资金流/订单流/跨品种联动等差异化特征。
2. **FT-Transformer 早停较快**: 31个epoch就停了, 说明特征信号有限。但融合MAE (0.0563) 低于任何单模型, 证明 Transformer 学到了 GBDT 没学到的信息。
3. **暴涨后空仓期**: 模型学到"暴涨后短期回调"模式, 2025年8月清仓2个月, 错过横盘收益。这是风控代价 (同期躲过了19%回撤)。
4. **单标的风险**: 只交易一只 ETF, 无法分散。可扩展 BUY_LIST 支持多标的轮动。
